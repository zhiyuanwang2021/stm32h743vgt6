## Project Overview

STM32H750VBTx firmware for a data acquisition device ("实验机"). The MCU reads voltage from a CS5552 24-bit ADC via SPI3, applies a sliding-window average filter, and outputs readings over UART1.

## Build

Build from Keil MDK-ARM IDE with ARM Compiler 5 (ARMCC V5.06 update 7). Project file: `MDK-ARM/shiyanji_H7.uvprojx`.

- **Target**: STM32H750VBTx (Cortex-M7, FPU)
- **Defines**: `USE_PWR_LDO_SUPPLY`, `USE_HAL_DRIVER`, `STM32H750xx`
- **Output**: `MDK-ARM/shiyanji_H7/shiyanji_H7.axf` + `.hex`
- **Debug probe**: ST-Link

No CLI build; compile and flash via Keil IDE.

## Architecture

```
STM32CubeMX generated (HAL)          User code
─────────────────────────────      ─────────────
Core/Src/main.c                    Core/Src/user.c      — CS5552 driver + LED
Core/Src/stm32h7xx_it.c            Core/Src/filter.c     — sliding avg filter
Core/Src/stm32h7xx_hal_msp.c       Core/Inc/user.h       — CS5552 API + pin defines
Core/Src/*.c (peripheral init)     Core/Inc/filter.h     — filter API
Drivers/ (HAL + CMSIS, read-only)
```

**User code is confined to `Core/Src/` and `Core/Inc/`.** Everything under `Drivers/` is vendor HAL/CMSIS — treat as read-only.

### Key files

| File | Role |
|---|---|
| `Core/Src/user.c` | CS5552 driver (SPI reg read/write, single/continuous conversion, calibration, voltage conversion) + LED control |
| `Core/Src/main.c` | `main()` — sys init, peripheral init, CS5552 init, main loop polls ADC every 100ms |
| `Core/Src/filter.c` | 10-sample sliding-window average filter |
| `Core/Inc/user.h` | CS5552 register map, command codes, pin macros, public API |
| `Core/Inc/main.h` | All GPIO pin definitions (generated by CubeMX) |

### CS5552 driver (user.c) — key details

- **SPI3** used for CS5552 communication
- Chip select: `P_ADCCS2` (PD3)
- SDO readback pin: PC11
- Reference voltage: 2500mV (CS5552_VREF_MV)
- Register access: 1-byte command (with even parity) + 4-byte data (32-bit mode via `CS5552_REG_24BIT_MODE=0`)
- Single conversion mode: `CS5552_ReadADC_Single()` — send command, wait SDO low→high→low, read 32-bit frame
- Continuous conversion: `CS5552_StartContConv()` keeps CS low, then `CS5552_ReadADC()` polls data
- `printf()` outputs to UART1 via `fputc()` override in main.c

### Other peripherals active

- W5500 Ethernet (SPI), MRAM (SPI), DAC (SPI) — pins defined in main.h but logic not in user code yet
- 3× SPI, 2× USART, 1× I2C, many TIM channels

## Code conventions

- STM32CubeMX generated code uses `/* USER CODE BEGIN/END */` markers — place new code only within these blocks
- New user modules go in `Core/Src/` and `Core/Inc/`, separate from HAL glue
- Pin definitions belong in `main.h` (CubeMX managed)
- CS5552 driver uses `bool` return values; `true` = success
